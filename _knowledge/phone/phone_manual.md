# Phone Order Agent マニュアル — 電話

> 前提: 電話チャネル固有のルール。Phone Order Agentが参照する。

## 1. 終話・挨拶の判定

電話では注文の途中や最後に挨拶表現が含まれることがある。
以下のルールで「終話（farewell）」か「注文の続き」かを判定すること。

### 終話と判定するケース

発話の**主旨が挨拶のみ**で、商品名・数量・注文指示を含まない場合。

| 発話例 | 判定 |
|---|---|
| 「ありがとうございます」 | 終話 |
| 「失礼します」 | 終話 |
| 「以上です」 | 終話 |
| 「それだけです」 | 終話 |
| 「じゃあお願いします」 | 終話 |
| 「よろしくお願いします」 | 終話 |
| 「では失礼いたします」 | 終話 |

### 終話と判定しないケース

商品名・数量・注文指示など、注文に関する情報を含む場合は**注文として処理**する。

| 発話例 | 判定 | 理由 |
|---|---|---|
| 「ありがとうございます、りんごも追加で」 | 注文 | 商品の追加指示を含む |
| 「じゃあバナナ20kgお願いします」 | 注文 | 商品名と数量を含む |
| 「以上です、あ、やっぱりみかんも」 | 注文 | 追加指示を含む |
| 「よろしくお願いします、配送は午前で」 | 注文 | 配送条件の指示を含む |

### 判定の原則

- **迷ったら注文として処理する**（終話の誤判定は注文の取りこぼしにつながるため）
- 挨拶と注文指示が混在する場合は、注文指示を優先する

## 2. 終話時の出力形式

終話と判定した場合は、以下のJSON形式で返すこと。

```json
{
  "is_farewell": true,
  "response": "ありがとうございます。またのご注文をお待ちしております。失礼いたします。"
}
```

### 終話応答のバリエーション

状況に応じて適切な応答を選ぶこと。

| 相手の発話 | 応答例 |
|---|---|
| 「ありがとうございます」 | 「ありがとうございます。またのご注文をお待ちしております。」 |
| 「失礼します」 | 「失礼いたします。ありがとうございました。」 |
| 「以上です」「それだけです」 | 「かしこまりました。ご注文ありがとうございます。失礼いたします。」 |
| 「じゃあお願いします」 | 「かしこまりました。ご注文ありがとうございます。」 |
