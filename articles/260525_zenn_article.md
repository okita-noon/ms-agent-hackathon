## foogent（フージェント）とは

:::message
**foogent** は、**食品卸・食材メーカーの受注担当者**のための AI エージェントです。

電話・LINE・メールなどバラバラなチャネルに届く注文を自動で受け取り、在庫照合・受注確定・顧客への返信までを一気通貫でサポートします。担当者がやることは、AI が準備した受注確認内容を **ワンタップで承認するだけ**。
:::

| | 従来の受注業務 | foogent 導入後 |
|---|---|---|
| **注文受付** | 電話・LINE・メールを個別に確認 | 全チャネルを自動で一元受信 |
| **在庫確認** | 担当者が手動で照合 | AI がリアルタイムで自動照合 |
| **データ入力** | スプレッドシートに手入力 | 自動構造化・登録 |
| **顧客への返信** | 担当者が都度作成・送信 | AI が返信案を生成・ワンタップ承認 |
| **所要時間** | 毎朝 3〜5 時間 | 確認作業 5〜10 分 |

## 朝6時から始まる受注担当者の1日

午前6時。まだ外が暗い時間に、食品卸の受注担当者は仕事を始めています。

LINEを開くと昨夜からのメッセージが20件以上。着信もある。メールも来ている。

内容はぜんぶ「注文」です。

> 「トマト20kg、今日の夕方に」
> 「昨日頼んだキャベツ、やっぱり15kgに変えて」
> 「卵20ダース追加お願いします！！」

これを手作業でスプレッドシートに転記して、在庫と照らし合わせて、配送リストを作る。始業の7時までに終わらせなきゃいけない。でも注文が届く手段はバラバラ——LINE、電話、メール、ときには口頭。

**1人あたり毎朝1〜2時間。チーム全体で3〜5時間が、毎日この作業に消えています。**

---

これは、一つの現場だけの話ではありません。

ハモック社の調査（2023年・食品小売・卸売業の受注担当者106名対象）によると、受注手段として**電話が50%・メールが55.7%**と、いまも半数以上の現場が複数のアナログチャネルで注文を受け続けています。受注業務の課題として**「注文書の手入力」を挙げた担当者は60%**にのぼります。

さらに、インフォマート社の調査（2023年・食品卸企業349名対象）では、**受注業務に1日2時間以上かかると答えた担当者が3割以上**。デジタル化を阻む最大の障壁として**「取引先がWeb発注を嫌がる」（47.4%）**が挙げられており、現場が変われない構造的な理由がここにあります。

「効率化したいのに、変えられない」——なぜ変えられないのか。

注文を送ってくる側は、**中学を卒業してから40〜60代になるまで、ずっと料理の道に没頭し、技術を磨き上げてきたベテランシェフの方々**です。仕込みの合間に、使い慣れたLINEや電話でパッと注文を入れる。そのスピード感こそが彼らにとっての正義であり、新しいシステムの操作を強要することは、職人としての時間を奪うことに他なりません。

**「顧客の行動は変えない。だけど、私たちの毎日の業務は自動化する」**

この一見矛盾する高い壁を、AIエージェントの力で越えるために開発したのが foogent です。

## 解決したかった課題

### ターゲット：DXのはざまにいる中小食品卸

大企業は専用の EDI システムを持っています。でも **10〜50名規模の食品卸・食材メーカー** はそれに対応できていない。「Excel と電話とLINEで何とかしている」現場が、まだたくさんあります。

### 現場の具体的な困りごと

- **注文チャネルがバラバラ**：電話・LINE・メール・口頭が混在し、どこに何の注文が来たかわからなくなる（食品業界調査で電話50%・メール55.7%と複数チャネルが並立）
- **手動転記でミスが起きる**：同じ商品名でも「トマト」「とまと」「tomato」と表記が揺れ、担当者の60%が手入力でシステムに登録している
- **在庫確認の往復コストが高い**：注文を受けるたびに在庫担当に確認し、返答して、を繰り返す
- **朝のピーク時に処理が追いつかない**：注文が集中する時間帯に人手が足りず、漏れが発生する

**「全員の朝をもっと人間らしくしたい」——それが foogent の目指すものです。**

### 解決するとどうなる？

受注業務が自動化されることで：

- 担当者の確認作業が **毎朝3〜5時間 → 5〜10分** に短縮
- 空いた時間を **顧客との関係づくり・営業活動** に使える
- ミス・漏れが減り、**精神的負荷が大幅に下がる**

参考として、ERPと受発注システムの連携を進めた日清食品ホールディングスでは、**年間約2万4千時間分の工数削減**を実現しています。大企業での実績が示すように、受発注のデジタル化が生む価値は確実に存在します。中小規模の現場にも、同じインパクトを届けることが foogent の目標です。

## foogentとは

**foogent** = food × agent

食品卸・食材業界の受発注業務を、複数の AI エージェントが協調して自動一元管理する **マルチテナント対応 AI Agent SaaS** です。

### できること

| 機能 | 概要 |
|------|------|
| マルチチャネル注文受付 | LINE・電話・メールの注文を自動解析・構造化 |
| 在庫照合・欠品アラート | 注文受信と同時に在庫を確認、欠品は即通知 |
| ワンタップ承認 | AIが作った受注確認案を担当者がワンタップで承認 |
| 到着予定日の自動推定 | リードタイム・締め時間・得意先定休日を考慮 |
| 休眠顧客への販促支援 | 発注が途絶えた顧客を検知し、販促メッセージを自動生成 |
| 受注ダッシュボード | 全受注状況をリアルタイムで一元管理 |

## AIエージェントの設計

foogent の核心は、**複数の AI エージェントが協調して動く**設計です。単なる自動化ツールではなく、状況を判断して行動する「エージェント」として設計しました。

### エージェントの役割分担

```
📱 注文受付エージェント
  → LINE・電話・メールから注文情報を自然言語で抽出・構造化

📦 在庫照合エージェント
  → 在庫 DB と照合し、欠品・代替品の提案を判断

🖥️ ダッシュボードエージェント
  → 異常検知・トリアージ・担当者への優先度付き通知

📣 営業支援エージェント
  → 休眠顧客の検知・発注傾向に合わせた販促メッセージ生成
```

<!-- 📌【図を必ず挿入】エージェント間の連携フロー図（シーケンス図またはフローチャート）を挿入してください -->

### なぜ「単なる自動化」ではなく「エージェント」なのか

従来のプログラムや単純なLLM呼び出しでは、「手書きの誤認識」や「キャビア 1pc」と「キャビア 1箱」のような表記ゆれに対応できませんでした。

foogent が「エージェント」と呼べる理由は、以下の**自律的な思考サイクル**を持っているからです。

1. **抽出（Perceive）：** バラバラな形式から注文内容（顧客名・商品名・数量・納品日）を読み取る
2. **推論（Think）：** 顧客マスターや過去の注文履歴を自ら「検索ツール」で参照し、「このシェフが言う『いつもの』は、この商品のことだな」と自律的に推論する
3. **変換（Act）：** 現場のルールに則り、グラム→キログラムの単位変換や加熱処理有無の整合性チェックをしてデータを整形する

この思考サイクルが、「受け取る→判断する→整形する」を人間なしで完結させます。

### Semantic Kernel を選んだ理由

Microsoft エコシステムとの親和性が高く、**複数エージェントの協調制御**を実装しやすいライブラリです。プランナー機能でエージェント間のタスク分解・委譲を実現し、会話履歴を保ちながら自然な注文受付フローを実現しました。

### 注文解析の実際

LINE から来るメッセージはこんな感じです：

> 「今日の夕方にトマト20kg追加で！あと、昨日頼んだキャベツはやっぱり15kgに変更で」

これを受け取った foogent は：

1. 品目・数量・配送時間帯を抽出
2. 「変更」「追加」「キャンセル」を判別
3. 既存注文との差分を検出
4. 在庫照合 → 受注確認メッセージを自動生成

担当者はAIが作った確認メッセージを確認してワンタップ承認するだけ。

### 電話チャネルへの対応

Azure Communication Services Call Automation を使って電話注文にも対応。通話内容をリアルタイムでテキスト変換し、同じパイプラインで処理します。「電話してきた人だけ損をする」状況をなくしました。

## 実装で工夫したこと

### 1. Human-in-the-Loop 設計

AIの認識率が100%でない限り、誤発注は命取りです。そのため foogent は、エージェントが「確信度が低い」と判断した注文データだけをピックアップし、人間がUI上で1クリックで修正・確定できる設計にしました。

問題のない注文はそのまま自動処理。疑わしいものだけが人間の目に届く——この仕組みで、**AIへの信頼と人間の安心を両立**しています。

### 2. 会話履歴の保持（Cosmos DB）

「昨日頼んだ〇〇を変更したい」という自然な会話に対応するため、Cosmos DB に会話セッションを保存しています。コンテキストを持った注文変更・キャンセルが可能で、「どの注文か」を担当者が都度指定しなくて済みます。

### 3. 到着予定日の自動推定

「今日の夕方注文」でも「それは明日の午前着になります」と即答できる機能を実装しました。リードタイム・注文締め時間・得意先ごとの定休日を考慮して自動で推定しています。

<!-- 📌【スクリーンショット】到着予定日の推定が表示されている画面を挿入してください -->

### 4. 商品名・顧客名の表記ゆれ対応

「トマト」「とまと」「tomato」——同じ商品でも担当者によって表記が違います。LLM を使った標準化 + 商品マスタとの照合で、表記ゆれによる転記ミスを防いでいます。

### 5. 休眠顧客への販促支援

一定期間注文がない顧客を自動検知し、その顧客の発注傾向に合わせた販促メッセージを AI が生成。担当者が確認・承認してから送信する形で、営業活動を後押しします。

<!-- 📌【スクリーンショット】休眠顧客リストと販促メッセージ生成画面を挿入してください -->

### 6. マルチテナント設計

複数の食品卸事業者が使える SaaS 設計。JWT 認証でテナントを分離し、データの混在を防いでいます。

## まとめ

### foogent の AI エージェントとしての特徴

foogent は、担当者が「承認ボタン」を押すまで、すべての処理をバックグラウンドで自律的に完結します。

電話・LINE・メールを問わず届いた注文を自動で受け取り、在庫を照合し、返信案を生成する——この一連の流れに、担当者の操作は不要です。従来の「ユーザーが質問して、AI が答える」対話型エージェントとは異なり、**状況を察知して自ら動く**一歩進んだ AI エージェントとして機能しています。

注文受付・在庫照合・ダッシュボード・営業支援の 4 つのエージェントが協調することで、単一のチャットボットでは対応できなかった**業務フロー全体**をカバーしています。

### foogent の技術的な特徴

Microsoft Azure の各種サービスを活用し、運用面・拡張性に優れたシステム構成となっています。

これらを組み合わせることで、**「作れる」だけでなく「運用できる」**プロダクトを目指しました。

---

今回、Microsoftの強力なAIエコシステムを活用したことで、これまで「職人の聖域」として自動化が不可能だと思われていた食品流通の受注業務に、エージェントという形で一石を投じることができました。

foogent が毎朝の4〜5時間を数分に短縮する未来。それは、単なるコスト削減ではなく、**日本の食を支える人々が、もっと誇りを持って、もっとクリエイティブに働ける時間を創り出す未来**です。

AI は「0から1」を生み出す存在ではなく、人間の「1」を「90」に引き上げる存在です。foogent が自動化した時間を、担当者が本来やるべき**顧客との関係づくり**に使える——そんな現場を、食品業界から作っていきたいと思っています。
