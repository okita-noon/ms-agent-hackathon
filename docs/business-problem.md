# 業務課題と解決策

## foogentが解決する業務課題

### 対象業種

食品卸・食材メーカー（中堅〜中小規模）

---

## 課題1: 受注チャネルが分散していて一元管理できない

### 現状の痛み

食品卸の現場では、顧客からの注文がさまざまなチャネルで届きます。

| チャネル | 課題 |
|---|---|
| **電話** | 音声のため記録が残らない。書き漏らし・聞き違いが発生する |
| **LINE** | 個人スマートフォンに届き、担当者不在時に見落とされる |
| **FAX** | 手書きのため読み取りに時間がかかる。デジタル化されない |
| **メール** | 受信確認が個人任せ。システム連携が難しい |

受注内容は担当者が手動でExcelや基幹システムへ転記する必要があり、**転記ミス・入力漏れ・重複受注**が常態化しています。

### foogentの解決策

すべてのチャネルからの注文を **AI Agentが自動受信・構造化** し、単一のダッシュボードに集約します。

```
LINE / 電話 / メール / FAX
        ↓
  AI Agentが自動処理
        ↓
  受注ダッシュボード（一元管理）
```

---

## 課題2: 曖昧な注文表現への対応に時間がかかる

### 現状の痛み

食品卸の顧客（飲食店・小売店）は日常的な口語で注文します。

- 「いつもの量で」
- 「りんご、多めに」
- 「ツナ缶100g」（缶の種類が複数存在する）
- 「明日の朝イチで」

これらの曖昧表現を解釈するために、担当者が顧客に電話折り返しする手間が生じています。

### foogentの解決策

**パターン学習AI**が過去の発注履歴を学習し、曖昧表現を自動解釈します。

- 「いつもの」→ 直近パターンを参照して自動確定
- 初回注文 or 曖昧な場合 → AIが確認メッセージを自動送信
- 学習が進むほど自動確定率が向上（継続的改善）

---

## 課題3: 誤発注の発見が遅れる

### 現状の痛み

顧客が数量を間違えて注文するケースが頻繁に発生します。

- 「15kg」のつもりで「150kg」と入力
- 「10箱」のつもりで「10kg」と送信

これらのミスは、ピッキング作業や出荷直前まで気づかれないことがあります。

### foogentの解決策

**Exception Agent**が注文を受信した時点でリアルタイムに異常検知します。

- 過去の注文パターンとのZ-score比較（統計的異常検知）
- 単位の不一致検知
- 異常を検知した場合、AIが即座に顧客へ確認メッセージを送信
- ダッシュボードの「例外ケース」パネルで担当者も確認可能

---

## 課題4: 担当者が在席しないと受注処理が止まる

### 現状の痛み

- 朝の忙しい時間帯に注文が集中するが、担当者は配送手配で手が離せない
- 休暇・病欠時に注文を見落とす
- 夕方以降の注文は翌日対応になる

### foogentの解決策

AI Agentが **24時間365日** 受注処理を代行します。

- 在庫確認・引当まで自動実行
- 問題なければ自動確定・顧客への返信まで完結
- 要対応案件のみ担当者にエスカレーション

---

## システムが生み出す価値

| 指標 | 改善効果（想定） |
|---|---|
| 受注入力工数 | 約70%削減 |
| 誤発注・転記ミス | ほぼゼロ（AI自動検知） |
| 受注処理リードタイム | 最大8時間 → 数秒 |
| 担当者の対応可能時間 | 24時間365日対応 |

## デモシナリオ

審査員向けデモでは以下の4シナリオを実演します。

### シナリオ1: 通常注文（LINE）

```
顧客: りんご10箱、バナナ20kgお願いします。明日の午前中で。
AI:  りんご10箱（冷蔵）、バナナ20kg（常温）、明日午前中お届けします。
```

→ ダッシュボードに「受注済み」として即反映

### シナリオ2: 曖昧表現の自動解釈

```
顧客: いつものお願いします。
AI:  （過去パターンを参照）りんご10箱（冷蔵）、明日お届けします。
```

→ 2回目以降は確認なしで自動確定

### シナリオ3: 誤発注検知

```
顧客: トマト150kgください。
AI:  トマト150kgですが、いつもは15kg前後のご注文です。数量をご確認いただけますか？
```

→ ダッシュボードの例外パネルに「数量異常」として表示

### シナリオ4: 在庫不足対応

```
顧客: りんご50箱お願いします。
AI:  りんごは現在30箱の在庫です。30箱でよろしいですか？または代替品をご提案できます。
```

→ 担当者は「要対応」としてダッシュボードで確認・Resolution Agentのプレビューを承認
